Creates a new Hypermerge instance that manages a set of documents. All previously opened documents are automatically re-opened.

Parameters
options (object)
Name Description
options.storage object config compatible with Hypercore constructor storage param
options.immutableApi boolean (default false) whether to use Immutable.js Automerge API
options.port number (default 0) port number to listen on
defaultMetadata (object = {}) default metadata that should be written for new docs

create

Creates an Automerge document backed by a new Hypercore.

If metadata is passed, it will be associated with the newly created document. Some metadata properties are assigned automatically by Hypermerge:

  • docId: An id for this document. Forking a document creates a new docId.
  • groupId: An id for this group of documents. Forking a document keeps the groupId.
create(metadata: object)
Parameters
metadata (object = {}) metadata to be associated with this document

fork

Creates a new actor Hypercore feed and Automerge document, with an empty change that depends on the document for another actor. The metadata of the new document will contain a parentId property.

fork(parentId: string)
Parameters
parentId (string) id of document to fork

merge

Takes all the changes from a document (sourceId) and adds them to another document (destId). Returns the merged document.

The source and destination docs must have come from the same root document. e.g. The source doc was a .fork() of the destination doc, or visa-versa.

merge(destId: string, sourceId: string)
Parameters
destId (string) docId to merge changes into
sourceId (string) docId to copy changes from

delete
